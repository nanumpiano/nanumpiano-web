## 작성 규칙

1. 최신 작업을 항상 맨 위에 쓴다.
2. 제목에는 날짜, 시간, 작업자를 반드시 쓴다.
3. 실패 기록을 삭제하지 않는다.
4. 작업 중단 이유와 남은 문제를 반드시 남긴다.
5. 검증이 없으면 완료라고 쓰지 않는다.
6. Git 상태와 커밋 해시가 있으면 반드시 남긴다.
7. 외부 도구가 작성한 기록도 사용자 검토 전까지 삭제하지 않는다.
